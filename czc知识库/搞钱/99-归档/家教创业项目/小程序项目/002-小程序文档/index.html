注意事项

家长吐槽

分享一些来自家长的吐槽点

在过往的交流中，来自家长的怨言最集中的有以下几点:
1、迟到或卡着点到（包括在线辅导也存在迟到现象）。
2、辅导作业有错误。
3、辅导期间，辅导老师玩手机以及电子产品（如电子书或平板）。
4、辅导没有条理和控制力，容易被孩子带着走。
5、上门辅导空着手，让人感觉没有准备。
6、口气严重。虽然讲课讲得很好，但是口气太重,多次辅导后，学生由于无法忍受，要求换辅导老师。
7、辅导小学生多数是要辅导全科的，数学、英语的辅导，对很多辅导老师来讲，通常没有障碍。最有问题的是语文阅读理解和作文。一部分辅导老师在遇到情况之后，回去会有意识地准备，一部分每次都含糊过去，最终导致家长要换人。建议辅导老师在这方面多注意。
8、辅导过程中只顾赶作业，没有注意学习习惯的培养。家长找家教除了检查作业，解答错题之外，还希望能注重学习习惯的培养。
9、辅导没有计划，未能充分利用时间。如果遇到学生作业少的时候，早早写完作业，老师就不知道该做什么了。
10、有些家长，会在辅导间隙给辅导老师一些水果点心之类的吃。但是能不能要，辅导老师要自己拿捏。我们曾经接到投诉，对方的理由之一就是给辅导老师东西，辅导老师就吃了，还吃完了一盘。这是真事。我们不便有什么评价，只能陈述事实。
11、着装问题：脏辫，浓妆，着装暴露等。偶尔一次还能理解，可能课后有别的活动，多次这样的话，容易被辞退。对于来自学生家长的这些吐槽之处，我们与其有情绪来争辩，不如想一想怎样避免这样的事情发生在自己身上。在穿着与化妆这一方面，抛开对学生的影响,女辅导老师更多要考虑的其实是对自己的保护。
12、最难的，莫过于人事，诸多点点滴滴，错综复杂，是与非往往没有明确的界限，要是较真起来，似乎各方都可以有理有据。每个人都会觉得自己是委屈的一方。有时候你会发现，并不存在吃亏以及占便宜的说法。付出一定有回报，好与坏，都是人，终究还是自己与自己的纠缠。不妨开阔一点,多审视自己。我们见过太多无助的家长，苦于孩子总不进步，觉得没有遇到有责任心的老师。也见过太多辛苦的辅导老师，用心地做着每一件事，即使不再辅导，家长都会感叹难再找到这样的老师。其实大家都是过了高考这一关的人，能力都在，区别就是态度，如同备课、刷题、不迟到等等这些细节。

有的人一个学生能带三年，家长还介绍辅导朋友家的孩子。有的人则很快就被换掉。

我们都不易，无论怎样的天气，都有辅导老师有坐一个小时甚至半小时的车去做家教。有的人整个寒暑假都少有闲暇。然而社会就是社会，仅仅是汗水，并不一定就能获得尊重。抛开运气的成分，尊重往往是赢得的。只有真的用心，才能赢得尊重。

愿我们的每一份付出，都能得到善待。愿我们不辜负每一颗怀有期望的心。愿我们能尊重辅导这项兼职工作，也为自己赢得尊重。

前路漫漫，道阻且长，与君共勉!

优秀老师心得

分享优秀辅导老师的一些细节

1、有备课
2、有买课本以及辅导书的习惯。有很多人每接单一份家教后，就迅速买了课本和书籍。这样确实能让以后的家教过程更节省时间。
3、有刷题的习惯，准备充足，辅导的时候更得心应手。
4、每次家教结束之后都会跟家长当面，或短信，微信等交流一下，说一下学生当天的情况。

无论是准备教材或辅导书，还是备课，以及刷题，其实占用不了多少时间和精力，但是有准备和没有准备，差别还是很明显的。

这是一份兼职，即工作，在大学生所有的兼职工作中，家教这份工作，相对来说还是比较体面，且回报也比较可观的一种。

希望大家能珍惜。

二、防跳单与信任机制落地

1. 试课担保流程（关键！）

  1. 家长在小程序支付试课费（例如100元） → 平台冻结款项

  2. 教师接单并完成试课 → 家长24小时内确认试课结果

  3. 若满意：试课费转为正式课时费（教师可提现，平台抽成10%）

  4. 若不满意：平台全额退款，并免费重新匹配教师

2. 跳单预防设计

  • 隐性约束：

    • 教师接单需缴纳100元信用保证金（可退），若被发现跳单则扣除；

    • 家长端显示“通过平台续费可享受投诉优先处理权益”。

  • 显性约束：
    用户注册时勾选协议：“私下交易需赔偿平台3倍佣金”（法律威慑为主，实际执行成本高）。

3. 信任背书

  • 教师认证：
    大学生需上传学生证+成绩单，在职教师需教师资格证

  • 学习报告：
    教师每5次课需提交简要教学总结（小程序端30字+图片），家长可随时查看

五、风险与应对

  • 教师质量不稳定：
    设立“教师考试”（例如上传5分钟试讲视频，由平台审核）

  • 家长过度退款：
    设定“每位家长仅限2次免费退款”，后续退款扣10%手续费

  • 竞品模仿：
    快速建立区域化壁垒（例如专注海淀区，与本地托管机构合作）

最后建议

  1. 先做小程序，暂不做APP：微信生态足以支撑初期用户

  2. 严格把控教师审核：前100名教师亲自电话面试，确保口碑

  3. 坚持“单点突破”：初期只做小学语数外，拒绝过早扩张科目

二、开发文档（核心模块）

1. 技术选型

模块 技术方案 说明
前端开发 Uni-App + Vue3 跨平台兼容，节省开发成本
后端服务 腾讯云开发（CloudBase） 无需自建服务器，集成数据库、云函数
支付接口 微信支付Native API 需企业资质，费率0.6%
即时通信 小程序内置客服消息 避免跳转微信私聊，保留平台沟通记录

2. 数据库设计（核心表结构）

表1：用户表（user）

字段名 类型 说明
_id string 用户ID（自动生成）
role string 角色（student/teacher）
phone string 绑定手机号
audit_status int 审核状态（0未审/1通过/2驳回）

表2：教师表（teacher）

字段名 类型 说明
user_id string 关联user._id
certificate array 证件图片URL列表
subjects array 可授科目（["数学","英语"]）
score float 评分（默认5.0）

表3：订单表（order）

字段名 类型 说明
student_id string 学生用户ID
teacher_id string 教师用户ID
status string 状态（pending/success/failed）
deposit float 试课押金金额

3. 接口定义（示例）

接口1：提交教师审核

  • 路径：/api/teacher/audit

  • 方法：POST

  • 参数：

    {  
      "userId": "123",  
      "certificateUrls": ["url1", "url2"],  
      "subjects": ["数学"],  
      "price": 80  
    }
  • 响应：
    { "code": 200, "msg": "提交成功，预计24小时内审核" }

接口2：试课押金支付

  • 路径：/api/order/create

  • 方法：POST

  • 参数：

    {  
      "studentId": "456",  
      "teacherId": "123",  
      "amount": 100  
    }
  • 响应：
    { "code": 200, "data": { "paymentId": "TX20240324123456" } }

4. 关键功能实现逻辑

试课担保流程：

  1. 家长支付押金 → 生成预订单（状态pending）

  2. 教师接单 → 平台推送微信服务通知

  3. 试课完成 → 家长确认 → 平台释放款项（教师可提现）

  4. 若超时未确认 → 自动退款至家长账户

防跳单逻辑：

  • 教师提现时校验订单状态，若存在跳单记录（如同一家长多次线下交易），冻结账户
